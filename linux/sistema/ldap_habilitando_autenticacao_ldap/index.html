    Habilitando a autenticação LDAP no Ubuntu 20.04¶

    Instalando os pacotes¶

    sudo apt-get update               
sudo apt-get install -y libnss-ldap ldap-auth-client libpam-ldap ldap-utils nscd

    Obs.: O pacote nscd, não influencia na configuração do ldap, porém ele é importante pois consulta a base constantemente mantendo atualizado nas máquinas informações como alterações de senha por exemplo.

    Ao executar o instalador será necessário incluir as informações:

    • URI : ldaps://sldap.lncc.br
    • Base : dc=lncc,dc=br
    • Ldap_version : 3
    • pam_password : md5

    Perguntas do instalador:

    • LDAP server Uniform Resource Identifier : ldaps://sldap.lncc.br
    • Distinguished name of the search base : dc=lncc,dc=br
    • LDAP version to use : 3
    • Make local root Database : No
    • Does the LDAP database require login : No

    OBS:

    Por padrão, vamos utilizar o protocolo ssl

    DICA:

    Caso precise reconfigurar o cliente LDAP edite o arquivo /etc/ldap.conf ou utilize o comando :

    dpkg-reconfigure ldap-auth-config

    Configurar o Service Switch (NSS)¶

    Devemos ajustar o arquivo /etc/nsswitch.conf. Ele define a ordem do processo de autenticação.

    sudo nano /etc/nsswitch.conf

    Ajuste as seguintes linhas: 

    #passwd:         files systemd
#group:          files systemd

passwd:         files systemd ldap
group:          files systemd ldap
shadow:         files

    Configurar o PAM¶

    Utilizamos o comando abaixo para configurar o PAM para utilizar a autenticação do ldap. 

    sudo pam-auth-update

    Na tela apresentada, marque todas as opções. Com a opção create home directory on login o sistema criar automaticamente o Home Dir dos usuários quando ocorrer o primeiro login.

    Evitar consulta ao LDAP¶

    Para que usuários do sistema não sejam consultados no ldap adicionaremos ao arquivo de configuração (/etc/ldap.conf) algumas exceções: 

    sudo nano /etc/ldap.conf

    Cole a linha abaixo no final do arquivo: 

    nss_initgroups_ignoreusers backup,bin,daemon,games,gnats,irc,landscape,ldap,libuuid,list,lp,mail,man,messagebus, news,ntp,openldap,proxy,root,sshd,statd,suporte,sync,sys,syslog,uucp,whoopsie,www-data

    OBS:

    Adicionamos ao arquivo os logins utilizados por alguns programas padrão, mas podemos inserir, nesta lista, o login de usuários de acordo com a necessidade.

    Configurar SSL¶

    Abra o arquivo /etc/ldap/ldap.conf para edição

    sudo nano /etc/ldap/ldap.conf

    E adicione a seguinte linha o final do arquivo:

    TLS_REQCERT never

    Testanto a configuração¶

    Pare o serviço nscd

    sudo systemctl stop nscd

    Tente obter os IDs de um usuários

    id giselecz

    A resposta deve ser:

    uid=30251(giselecz) gid=2002(helpdesk) grupos=2002(helpdesk),
999(admin),512(Domain Admins),2170(adminsuporte)

    Reinicie o serviço nscd: 

    sudo systemctl start nscd
sudo systemctl status nscd

    DICA :

    Caso precise reconfigurar o processo de autenticação, utilizando o modo avançado, utilize o comando abaixo:

    sudo dpkg-reconfigure ldap-auth-config

    Configurando o Home do usuário¶

    Em máquinas onde usaremos o disco local é necessário criar o diretório /prj. Assim quando ocorrer o primeiro login do usuário o sistema irá criar o seu Home Dir de acordo com o diretório informado pelo LDAP.

    sudo mkdir /prj

    DICA :

    Se após a configuração acima, nos sistemas baseados no Ubuntu Linux, não possível loga-se no modo gráfico, será necessário configurar o arquivo /etc/lightdm/lightdm.conf

    sudo vim /etc/lightdm/lightdm.conf

    Adicione as seguintes linhas: 

    [SeatDefaults]
greeter-show-manual-login=true
autologin-guest=false
autologin-user=
autologin-user-timeout=0
autologin-session=lighttdm-autologin
greeter-session=unity-greeter

    Caso a configuração do LDAP esteja sendo efetuada nos laboratórios ou máquinas públicas, na etapa 6 desse procedimento deve-se montar o projeto nas máquinas, para que o home do usuário acessível seja o da rede.

    Os usuários que precisam ter acesso a maquina via SSH devem ser adicionados manualmente no grupo admin. Podemos utilizar o comando abaixo para adicionar os usuários ao grupo:

    sudoadduser [login_name] admin

    Ou podemos adicionar a lista de todos os usuários, diretamente no arquivo /etc/groups utilizando o comando:

    sudo vigr

    E ajustando a linha abaixo: 

    admin:x:1001:luisrodrigoog,lrodrigo,izabel,giselecz,andrerc,suporte,brunoaf,cdaniel,camilags,carneiro,fabio,eloirgst

    Com isto finalizamos o processo de configuração da autenticação baseada no LDAP
